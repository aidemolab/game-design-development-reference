# 16. Game Design Checklist

## Concept

- [ ] Can the concept be explained in two sentences?
- [ ] Is the player fantasy clear?
- [ ] Is the intended audience identified?
- [ ] Is the game original in identity and expression?

## Player experience

- [ ] Is the objective clear?
- [ ] Is the core loop repeatable and rewarding?
- [ ] Are decisions meaningful?
- [ ] Are success and failure understandable?

## Components

- [ ] Player and controls
- [ ] Camera
- [ ] Environment and levels
- [ ] Objectives, collectibles, enemies, or strategic entities
- [ ] Physics and collision
- [ ] Game states and flow
- [ ] UI and feedback
- [ ] Scoring, economy, health, energy, or time
- [ ] Progression and difficulty
- [ ] Audio, effects, animation, and lighting
- [ ] AI and opponent behaviour
- [ ] Saving, restart, and completion

## Scope

- [ ] Is the first playable version defined?
- [ ] Is every first-version feature essential to testing the core loop?
- [ ] Have multiplayer, persistence, procedural generation, and advanced AI passed a scope gate?
- [ ] Can the milestone be verified and committed independently?

## Evidence and ethics

- [ ] Are sources cited?
- [ ] Have licences been checked?
- [ ] Are code rights separated from asset rights?
- [ ] Are transferable principles separated from protected expression?
