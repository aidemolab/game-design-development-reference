# 10. UI, Feedback, and Accessibility

## Standard

Players should know the objective, current state, available action, consequence, and next step. UI should communicate necessary information without obscuring play.

## Best practices

- Prioritise information by urgency and frequency.
- Use consistent placement, terminology, colours, icons, and input prompts.
- Provide feedback through more than one channel when possible.
- Support text scaling, contrast, remapping, subtitles, volume controls, and reduced-motion needs where appropriate.
- Explain failure and recovery.
- Test UI at different resolutions and aspect ratios.

## Reference lessons

Boss Room demonstrates UI integrated with connection, selection, gameplay, and post-game states. Red Runner shows the simpler needs of scoring and restart. Strategy games demonstrate the challenge of presenting resources, production, technology, units, territory, and alerts without overwhelming the player.

## Review questions

- What information is essential every second?
- What can appear only when relevant?
- Can the player understand an icon without memorising it?
- Does colour have a text, shape, sound, or positional alternative?
