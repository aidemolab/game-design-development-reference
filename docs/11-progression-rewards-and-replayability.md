# 11. Progression, Rewards, and Replayability

## Standard

Progression should create new choices, mastery, or context, not only larger numbers. Rewards should reinforce the behaviours the design values.

## Lessons

- Age of Empires uses technological eras as clear strategic milestones.
- Albion Online uses equipment-driven specialisation and persistent mastery.
- Mindustry combines unlocks, production complexity, maps, and optimisation.
- PUBG creates replayability through variable equipment, opponents, routes, and match pressure rather than permanent power alone.

## Best practices

- Distinguish player skill, character power, content access, and cosmetic expression.
- Make the next goal visible and achievable.
- Avoid rewards that invalidate earlier choices immediately.
- Use variation, alternative strategies, procedural combinations, or opponent behaviour to support replayability.
- Ensure failure still produces learning.

## Scope warning

Do not build a long progression system before the basic loop is enjoyable. A prototype can use one level, one upgrade decision, and a clear completion state.
