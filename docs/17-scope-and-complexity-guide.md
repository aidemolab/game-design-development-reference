# 17. Scope and Complexity Guide

## Standard

Scope is the number of interacting promises made to the player. A small game with polished interaction is a stronger first project than an unfinished imitation of a commercial game.

## Complexity levels

### Foundation

One player, one scene, simple controls, one objective, basic feedback, restart.

### First playable

Complete loop, clear success/failure, small UI, tuned controls, representative visuals and audio.

### Vertical slice

One polished segment demonstrating target quality, architecture, content pipeline, and major risks.

### Production expansion

Additional levels, enemies, progression, saving, accessibility, optimisation, and platform work.

### High-risk systems

Multiplayer, persistent economies, live service, advanced procedural worlds, complex RTS AI, user-generated content, cross-platform accounts, and anti-cheat.

## Scope-control questions

- What hypothesis does this feature test?
- Can the game work without it?
- Does it create new content, technology, UI, testing, and maintenance obligations?
- Can a simpler mechanic produce the same player experience?
- What will be postponed if this feature is accepted?

## Rule

Borrow ambition from successful games, not their production scale.
